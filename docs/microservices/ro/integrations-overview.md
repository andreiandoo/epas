## Documentație Tehnică

Toate integrările urmează pattern-uri consistente:

### Autentificare
- OAuth 2.0 pentru majoritatea serviciilor
- Autentificare cheie API unde e necesar
- Stocare credențiale criptată
- Reîmprospătare automată token

### Suport Webhook
Integrările care suportă webhookuri primesc actualizări în timp real pentru:
- Evenimente comandă
- Evenimente bilet
- Evenimente client
- Evenimente plată

### Exemplu Configurare

```php
'integrations' => [
    'slack' => [
        'client_id' => env('SLACK_CLIENT_ID'),
        'client_secret' => env('SLACK_CLIENT_SECRET'),
    ],
    'hubspot' => [
        'client_id' => env('HUBSPOT_CLIENT_ID'),
        'client_secret' => env('HUBSPOT_CLIENT_SECRET'),
    ],
    // ... mai multe integrări
]
```

### Pattern Endpoint-uri Comune

Fiecare integrare urmează un pattern API similar:

```
GET /api/integrations/{provider}/auth
POST /api/integrations/{provider}/callback
GET /api/integrations/{provider}/connection
DELETE /api/integrations/{provider}/disconnect
POST /api/integrations/{provider}/sync
GET /api/integrations/{provider}/status
```
